## Overview

## Bug Fixes

### Trivy Operator — Talos read-only filesystem (`CreateContainerError`)

**Root cause:** Trivy's `node-collector` DaemonSet tried to mount `/etc/systemd` and `/lib/systemd` as hostPath volumes. Talos has a read-only root filesystem — these paths do not exist, causing `failed to mkdir "/etc/systemd": read-only file system`.

**Fix:** Stripped the systemd and kubelet-standalone mounts from `nodeCollector.volumes` and `nodeCollector.volumeMounts`. Only k8s-relevant paths kept: `/var/lib/etcd`, `/var/lib/kubelet`, `/var/lib/kube-scheduler`, `/var/lib/kube-controller-manager`, `/etc/kubernetes`, `/etc/cni/net.d/`.

**File:** `workloads/security/trivy.go`

---

### OTel Collector — duplicate `K8S_NODE_IP` env var (ArgoCD `Unknown` sync)

**Root cause:** The `kubeletMetrics` preset in the OpenTelemetry Helm chart automatically injects `K8S_NODE_IP` via Downward API. Our code also explicitly added it via `extraEnvs`. Kubernetes rejected the manifest with duplicate env entries, causing ArgoCD to be unable to calculate the diff (`Unknown` sync status).

**Fix:** Removed the explicit `extraEnvs` block from the agent config. The preset-injected variable is sufficient — the `kubeletstats` receiver endpoint `https://${env:K8S_NODE_IP}:10250` still resolves correctly.

**File:** `workloads/observability/otel_collector.go`

---

### NetBird peer — worker1 nodeSelector removed

The `k8s-routing-peer` StatefulSet was pinned to `k8s-worker1` via `nodeSelector`. Worker1 going down took the WireGuard tunnel with it, making all public services unreachable. Removed the nodeSelector — the pod now floats to any healthy worker. The RWO PVC (Longhorn) follows via detach/reattach on the new node.

**File:** `workloads/networking/netbird_peer.go`

---

### Cilium — NIC device name change (`eth0` → `ens18`)

**Root cause:** Talos v1.13.3 renamed the primary NIC from `eth0` to `ens18` on cluster nodes. Cilium's `devices` config only listed `eth0`, causing CrashLoopBackOff on upgraded workers before the control plane was upgraded.

**Fix:** Set `devices: [eth0, ens18]` in `cilium.go` during the rolling upgrade window. Both names are listed so mixed-version nodes (some on v1.12.4, some on v1.13.3) all work. Once all nodes run v1.13.3, this can be simplified to `ens18` only.

**File:** `core/platform/cilium.go`

## Removals

### Rancher removed

Rancher was removed from CDK8s workloads. It added ~1000m CPU overhead per replica, caused double-reconciliation conflicts with ArgoCD for Fleet-managed apps, and is not needed for this cluster's use case (ArgoCD + Headlamp cover all management needs).

**Removed:**
- `workloads/management/rancher.go`
- `workloads/imports/rancher/` (CDK8s schema bindings)
- `workloads/imports/fleet/` (Fleet is a Rancher dependency — no longer needed)
- Rancher helm repo from `workloads/cdk8s.yaml`
- `management.NewRancherChart()` call from `workloads/main.go`
