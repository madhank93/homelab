## Bug Fixes

### Trivy Operator ‚Äî Talos read-only filesystem (`CreateContainerError`)

**Root cause:** Trivy's `node-collector` DaemonSet tried to mount `/etc/systemd` and `/lib/systemd` as hostPath volumes. Talos has a read-only root filesystem ‚Äî these paths do not exist, causing `failed to mkdir "/etc/systemd": read-only file system`.

**Fix:** Stripped the systemd and kubelet-standalone mounts from `nodeCollector.volumes` and `nodeCollector.volumeMounts`. Only k8s-relevant paths kept: `/var/lib/etcd`, `/var/lib/kubelet`, `/var/lib/kube-scheduler`, `/var/lib/kube-controller-manager`, `/etc/kubernetes`, `/etc/cni/net.d/`.

**File:** `workloads/security/trivy.go`

---

### OTel Collector ‚Äî duplicate `K8S_NODE_IP` env var (ArgoCD `Unknown` sync)

**Root cause:** The `kubeletMetrics` preset in the OpenTelemetry Helm chart automatically injects `K8S_NODE_IP` via Downward API. Our code also explicitly added it via `extraEnvs`. Kubernetes rejected the manifest with duplicate env entries, causing ArgoCD to be unable to calculate the diff (`Unknown` sync status).

**Fix:** Removed the explicit `extraEnvs` block from the agent config. The preset-injected variable is sufficient ‚Äî the `kubeletstats` receiver endpoint `https://${env:K8S_NODE_IP}:10250` still resolves correctly.

**File:** `workloads/observability/otel_collector.go`

---

### Fleet CDK8s import removed

Fleet is managed exclusively by Rancher (`apps.catalog.cattle.io`) and reinstalls automatically when Rancher is running. Managing it separately via CDK8s caused conflicts and double reconciliation. The `workloads/imports/fleet/` package is removed. Fleet remains healthy on the cluster ‚Äî Rancher manages its lifecycle.
